About the Project

Welcome to our Enterprise Development Experience project - Bed & Breakfast Management System. Developed by Brent Simons and Siebe Michiels, this project showcases our skills in enterprise-level application development using microservices architecture.

Project Overview

Our project, inspired by Brent's father's Bed and Breakfast, consists of four microservices - Bnb, Room, Reservation, and Person - each managing a specific aspect of the B&B operations. The microservices are powered by MySQL and MongoDB databases.

Purpose and Goals

We aim to create a comprehensive management system that enables the effective handling of multiple Bed & Breakfast establishments. Through our microservices architecture, we provide functionalities for managing rooms, reservations, and personnel, contributing to a seamless B&B experience.

Key Features

We've implemented CRUD operations for all microservices and crafted three interesting public endpoints. Ontop of that we made a gateway to unify and simplify interactions across our 4 microservices. Authentication is added using Google GCP authentication, using JWT tokens to safeguard access to the gateway and website.

Public Endpoints

GET /public/bnb?name=x is a public endpoint providing an overview of all our bnbs. Use the optional "name" parameter to filter bnbs by text. Example result includes names and room IDs.

GET /public/rooms/available?id=x is a public endpoint to check room availability. Required body includes roomCodes, startDate, endDate, and size. Example result includes roomCode and availability status.

GET /public/room/available?roomCode=x&months=x is a public endpoint to check a specific room's availability in the next month(s). Use roomCode and optional "months" parameter. Example result includes start and end dates.

Technologies Used

We utilized the Spring Boot (Framework) to create our microservices. Spring Framework is an open-source framework for building enterprise Java applications.

For our front-end we used Angular, a powerful framework for building dynamic web applications.

We employed Docker, a platform for automating the deployment and scaling of applications in containers. We also used github actions to automate the building of images and pushing them to docker Hub.
